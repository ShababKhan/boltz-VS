## 2024-05-19 - [Avoid N^2 tuple unpacks and lookups for upper triangle combinations]
**Learning:** Using `for i, j in zip(*np.triu_indices(n, k=1))` and checking `if i in map and j in map` iteratively creates pairs over the *entire* range of indices `[0..N) x [0..N)`, which scales poorly as `O(N^2)`, even if the actual keys present in `map` is a small subset. Also, iterating zip inside a Python loop involves a huge number of tuple unpacking and Python bytecode executions.
**Action:** When finding pairs of existing valid indices (like iterating over all pairs in `idx_map.keys()`), use `itertools.combinations(sorted(idx_map.keys()), 2)`. This iterates strictly over the elements that already exist in `idx_map`, operating natively and avoiding N^2 lookups for elements that are masked out.
