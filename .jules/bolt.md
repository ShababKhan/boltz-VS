## 2024-05-24 - Exception handling overhead in MMCIF parsing with Gemmi
**Learning:** In performance-critical sections like parsing large MMCIF files with `gemmi.cif.Block.find()`, relying on `contextlib.suppress(Exception)` to handle missing keys (which throws `IndexError`) is significantly slower than explicit length checks (`len(table) > 0`). Exception handling overhead in Python is costly when exceptions occur frequently, acting as an implicit control flow.
**Action:** Always prefer explicit bounds or membership checking before accessing elements in frequently called parsing loops. However, continue to use targeted exception handling (e.g., `try...except ValueError:`) for unpredictable data conversion errors like parsing strings containing missing value markers (e.g. `?` or `.`) into floats.
